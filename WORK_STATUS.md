# 드론방제앱 작업 진행판

마지막 갱신: 2026-06-10 01:34:34 +09:00

## 현재 목표

드론방제앱 MVP를 갤럭시에서 실제로 실행하고 현장에서 사용할 수 있는 수준까지 만든다.

중요 원칙:

- 작업 중간마다 검증한다.
- 사용량이 끝나기 전에 반드시 커밋하고 푸시한다.
- 실제 현장 흐름을 우선한다.
- 갤럭시 우선으로 진행하고, 웹은 화면 확인용 보조 수단으로 둔다.

## 현재 진척률

약 99%

## 완료된 핵심 작업

- Expo 기반 모바일 앱 기본 구조 생성
- Supabase Auth, Database, Realtime, Storage 연동 골격 작성
- 대시보드, 지도, 작업 목록, 관리자, 필지 상세, 필지 등록 화면 구성
- 작업 목록에 오늘/상태별 필터와 현장 요약 추가
- 필지 상세에서 내비게이션, 상태 변경, 사진 업로드 흐름 구성
- 필지 상세에서 작업 예정일, 메모, 완료 사진 목록을 확인하는 흐름 추가
- 필지 등록에서 농가 생성, 필지 생성, 기본 방제 작업 생성을 한 번에 처리
- 기존 농가 이름/전화번호 검색 후 선택해서 중복 농가 생성을 피하는 흐름 추가
- 필지 등록 시 담당 방제팀을 선택해 기본 방제 작업에 바로 배정하는 흐름 추가
- 관리자 화면에서 방제팀을 생성하는 흐름 추가
- 관리자 화면에서 방제팀 목록 확인과 담당자/연락처 수정 흐름 추가
- 관리자 화면에서 Supabase 환경변수와 핵심 테이블 연결 점검 흐름 추가
- 갤럭시 실기기 점검표에서 주요 MVP 흐름 완료 여부를 로컬 저장하는 흐름 추가
- 필지 등록에서 현재 위치로 중심 좌표를 입력하고 반경 기반 임시 polygon을 생성하는 흐름 추가
- Supabase 환경변수 미설정 상태에서도 샘플 데이터 화면으로 안전 진입
- 로그인 화면에서 Supabase 연결 전 샘플 데이터 둘러보기 가능
- 샘플 모드에서 작업 상태 변경이 대시보드, 목록, 상세 화면에 유지되는 흐름 추가
- 샘플 모드에서 사진 선택 후 완료 사진 목록과 썸네일을 확인하는 흐름 추가
- 샘플 모드에서 방제팀 생성/수정과 농가-필지-작업 등록/배정 저장 흐름 추가
- 샘플 모드 등록 데이터, 상태 변경, 사진 목록을 앱 재시작 후에도 로컬 유지
- 관리자 화면에서 샘플 데이터 초기화 후 반복 실기기 테스트 가능
- 새로 등록한 샘플 필지가 지도 선택/중심과 상세 화면 상태 변경/사진 저장에 바로 반영
- 갤럭시 실기기 점검표에 항목별 완료 시간과 전체 완료 안내 표시
- 갤럭시 실기기 점검표에서 위치/사진 권한과 Supabase 연결 모드 확인 가능
- 갤럭시 실기기 점검 결과를 리포트로 생성하고 공유 가능
- 갤럭시 실기기 점검 항목별 현장 메모를 저장하고 리포트에 포함 가능
- 갤럭시 실행 전 LAN 주소, `.env`, adb 상태를 확인하는 사전점검 명령 추가
- 구형/신형 갤럭시 사진 접근 권한 호환성 보강
- LAN QR 연결 실패 시 사용할 갤럭시 터널 실행 명령 추가
- 갤럭시 터널 실행용 `@expo/ngrok` 의존성 고정
- GitHub `jinsulchohae-drone-app` 저장소에 갤럭시 설치용 APK 릴리스 업로드 완료
- 갤럭시 APK 다운로드/설치/권한 문제 해결 문서 `APK_INSTALL.md` 추가
- 갤럭시 APK 릴리스 노트 `APK_RELEASE_NOTES.md` 추가 및 GitHub 릴리스 본문 보강
- Android/iOS 실제 지도와 웹 목록형 지도 미리보기 분리
- Expo SDK 의존성 정리
