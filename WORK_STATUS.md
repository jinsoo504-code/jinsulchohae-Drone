## 현재 목표

드론방제앱 MVP를 갤럭시에서 실제로 실행하고 현장에서 사용할 수 있는 수준까지 만든다.

중요 원칙:

- 작업 중간마다 검증한다.
- 사용량이 끝나기 전에 반드시 커밋하고 푸시한다.
- 실제 현장 흐름을 우선한다.
- 갤럭시 우선으로 진행하고, 웹은 화면 확인용 보조 수단으로 둔다.

## 현재 진척률

약 68%

## 완료된 핵심 작업

- Expo 기반 모바일 앱 기본 구조 생성
- Supabase Auth, Database, Realtime, Storage 연동 골격 작성
- 대시보드, 지도, 작업 목록, 관리자, 필지 상세, 필지 등록 화면 구성
- 작업 목록에 오늘/상태별 필터와 현장 요약 추가
- 필지 상세에서 내비게이션, 상태 변경, 사진 업로드 흐름 구성
- 필지 상세에서 작업 예정일, 메모, 완료 사진 목록을 확인하는 흐름 추가
- 필지 등록에서 농가 생성, 필지 생성, 기본 방제 작업 생성을 한 번에 처리
- 기존 농가 이름/전화번호 검색 후 선택해서 중복 농가 생성을 피하는 흐름 추가
- 필지 등록 시 담당 방제팀을 선택해 기본 방제 작업에 바로 배정하는 흐름 추가
- 관리자 화면에서 방제팀을 생성하는 흐름 추가
- 관리자 화면에서 방제팀 목록 확인과 담당자/연락처 수정 흐름 추가
- 필지 등록에서 현재 위치로 중심 좌표를 입력하고 반경 기반 임시 polygon을 생성하는 흐름 추가
- Supabase 환경변수 미설정 상태에서도 샘플 데이터 화면으로 안전 진입
- 로그인 화면에서 Supabase 연결 전 샘플 데이터 둘러보기 가능
- Android/iOS 실제 지도와 웹 목록형 지도 미리보기 분리
- Expo SDK 의존성 정리
- `npm run typecheck` 통과
- `npx expo-doctor` 18개 항목 통과
- 웹 번들 생성 확인
- Expo 웹 서버 실행 및 `localhost` 200 응답 확인
- 반복 검증 명령 `npm run check`, `npm run web:export` 추가
