# 갤럭시 Expo Go 실기기 테스트

마지막 갱신: 2026-06-10

## 실행 방법

1. PC와 갤럭시를 같은 Wi-Fi에 연결합니다.
2. 갤럭시에 `Expo Go` 앱을 설치합니다.
3. PC에서 `mobile-drone-app` 폴더로 이동합니다.
4. 아래 사전점검 명령으로 LAN 주소, `.env`, adb 감지 여부를 확인합니다.

```powershell
npm run galaxy:preflight
```

5. 아래 명령을 실행합니다.

```powershell
npm run start:galaxy
```

6. 터미널에 표시되는 QR 코드를 갤럭시 Expo Go로 스캔합니다.
7. 앱이 열리면 로그인 화면에서 `샘플 데이터로 둘러보기`를 누릅니다.

## 샘플 모드 점검 순서

1. 대시보드에서 오늘 작업 현황이 보이는지 확인합니다.
2. 지도 탭에서 필지를 선택하고 상세 화면으로 이동합니다.
3. 필지 상세에서 `방제 시작`, `방제 완료`, `문제 발생` 상태 변경을 확인합니다.
4. `사진 업로드`를 눌러 갤럭시 사진을 선택하고 완료 사진 목록/썸네일을 확인합니다.
5. 관리자 화면에서 방제팀을 생성하고 목록에서 수정합니다.
6. 필지 등록 화면에서 농가, 필지, 예정일, 담당팀을 입력합니다.
7. 현재 위치로 좌표를 입력하고 반경 기반 임시 필지를 생성합니다.
8. 저장 후 지도/작업목록/대시보드에 새 작업이 반영되는지 확인합니다.
9. 관리자 화면의 `갤럭시 실기기 점검표 열기`에서 완료 항목을 체크합니다.

## Supabase 실제 연결 점검

1. `.env.example`을 복사해 `.env`를 만듭니다.
2. `EXPO_PUBLIC_SUPABASE_URL`과 `EXPO_PUBLIC_SUPABASE_ANON_KEY`를 입력합니다.
3. Supabase SQL은 `supabase/schema.sql`을 적용합니다.
4. 앱을 다시 실행합니다.
5. 관리자 화면에서 `Supabase 연결 점검`의 `점검` 버튼을 누릅니다.
6. `farmers`, `fields`, `spray_teams`, `spray_jobs`, `spray_photos`가 정상으로 표시되는지 확인합니다.

## 현재 PC 환경 참고

- 이 PC에서는 `adb` 명령이 감지되지 않았습니다.
- 따라서 USB 연결 기반 `npm run android` 검증보다 Expo Go LAN 실행을 우선합니다.
- Android Studio/Platform Tools 설치 후에는 `adb devices`로 기기 인식 여부를 확인할 수 있습니다.
- `npm run galaxy:preflight`에서 LAN 주소가 표시되면 갤럭시와 같은 Wi-Fi 또는 같은 네트워크인지 비교합니다.

## 문제 발생 시

- QR 스캔이 안 되면 PC와 갤럭시가 같은 Wi-Fi인지 확인합니다.
- 회사/공유기 방화벽에서 LAN 접속이 막히면 모바일 핫스팟을 사용합니다.
- 앱이 오래된 화면을 보이면 터미널에서 `r`을 눌러 새로고침하거나 명령을 다시 실행합니다.
- 환경변수 변경 후에는 Expo 서버를 껐다가 다시 켭니다.
